  ことばの力のものさし・デジタル版 — アーキテクチャ詳細説明
================================================================================

■ 概要
------
本アプリは、文部科学省「ことばの力のものさし」（2025年）に基づき、児童生徒の
発話音声を AI で多角的に評価する Web アプリケーションです。

音声ファイルを入力として受け取り、3つの処理エンジン（音声信号処理・音声認識・
大規模言語モデル）を組み合わせて、「発達ステージ（A〜F）」と「習得ステップ
（1〜8）」の2軸を中心とした包括的な評価を行います。

■ ステップ1: 入力
================================================================================

【受け付ける入力】
  - 音声ファイル: .wav / .mp3 / .m4a 形式（最大 200MB）
  - 学年段階の選択: 小1〜小2 / 小3〜小4 / 小5〜中2 / 中3〜高校

【学年段階の役割】
  学年段階は「日本語の習得ステップ」の判定基準を切り替えるために使用します。
  同じ発話でも、小1〜小2と中3〜高校では判定基準（各ステップの到達目標）が
  異なるため、適切な学年段階の選択が正確な評価に直結します。

  STEP_RUBRIC 辞書に学年段階ごとのステップ記述が格納されており、LLM に送信
  するシステムプロンプトが動的に構築されます（build_system_prompt 関数）。

■ ステップ2: 音声信号処理（Librosa）
================================================================================

音声ファイルを直接解析し、テキスト化を介さずに取得できる定量指標を算出します。

【2-1. 流暢さ分析】 analyze_fluency()
  - librosa.load() で音声を波形データ (y, sr) として読み込み
  - librosa.effects.split(y, top_db=40) で有音区間を検出
    → ピーク音量から 40dB 以上静かな区間を無音と判定
  - 算出指標:
    - total_duration:   音声全体の長さ（秒）
    - speaking_duration: 有音区間の合計長さ（秒）
    - silence_duration:  無音区間の合計長さ（秒）
    - phonation_time_ratio: 発話時間比率（speaking / total）
  - 最低 0.5 秒以上の音声が必要（バリデーション）

【2-2. ポーズ分析】 analyze_pause()
  - 有音区間の間隔（ギャップ）を走査
  - 100ms 以上のギャップを「ポーズ」として認定
  - 算出指標:
    - pause_count:         ポーズの回数
    - mean_pause_duration: 平均ポーズ長（秒）
    - pause_rate_per_min:  1分あたりのポーズ頻度

【2-3. 韻律（イントネーション）分析】 analyze_prosody()
  - librosa.pyin() でピッチ（基本周波数 F0）を推定
    → pYIN アルゴリズム: 確率的 YIN に基づくロバストなピッチ推定
    → 検出範囲: C2（約65Hz）〜 C6（約1047Hz）
  - NaN（無声区間）を除外した有効なピッチ値から統計量を算出
  - 算出指標:
    - pitch_mean_hz:  平均ピッチ（Hz）
    - pitch_std_hz:   ピッチの標準偏差（Hz）
    - pitch_range_hz: ピッチの最大-最小範囲（Hz）

【2-4. エンゲージメント分析】 analyze_engagement()
  - librosa.feature.rms() で RMS エネルギー（声量の指標）を算出
  - RMS の変動係数（CV = std / mean）で声量のメリハリを定量化
  - ピッチの変動係数と組み合わせてエンゲージメントスコアを算出
  - スコア計算:
    - energy_score = clip(rms_cv / 0.5 * 50, 0, 50)
    - pitch_score  = clip(pitch_cv / 0.30 * 50, 0, 50)
    - engagement_score = energy_score + pitch_score（0〜100）
  - is_flat フラグ: rms_cv < 0.15 かつ pitch_cv < 0.10 の場合 True
    → 声量もピッチも平坦 = 不安・自信のなさの可能性を示唆

■ ステップ3: 音声認識（OpenAI Whisper）
================================================================================

音声をテキストに変換し、認識モデルの副次的情報も活用します。

【3-1. 書き起こし】 transcribe_audio()
  - Whisper base モデルを使用（@st.cache_resource でキャッシュ）
  - 日本語（language="ja"）に固定して認識
  - word_timestamps=True でトークン単位のタイムスタンプと確信度を取得
  - 戻り値:
    - text:        書き起こしテキスト全文
    - segments:    セグメント情報（タイムスタンプ付き）
    - token_probs: 各トークンの単語と確信度（probability）のリスト

【3-2. 発音自信度分析】 analyze_pronunciation_confidence()
  - Whisper の各トークンの probability（0.0〜1.0）を集計
  - 平均確信度（mean_confidence）を算出
  - 確信度 0.5 未満のトークンを「低確信度ワード」として抽出（上位10件）
  - 低確信度ワード = 発音が不安定な箇所 = 母語干渉の可能性がある箇所
  - confidence_score = mean_confidence * 100（0〜100 点）

【3-3. 発話速度分析】 analyze_speech_rate()
  - 書き起こしテキストの文字数 ÷ 発話時間（秒）で算出
  - 算出指標:
    - char_count:    文字数
    - chars_per_sec: 文字/秒

【3-4. 自己修復・フィラー検出】 analyze_self_repair()
  - 正規表現パターンで日本語のフィラーを検出:
    「えーっと」「えーと」「えー」「えっと」「あのー」「あの」
    「うーん」「うん」「んー」「まあ」「ええと」「そのー」「その」「なんか」
  - 算出指標:
    - filler_count:        フィラーの総出現回数
    - filler_examples:     検出されたフィラーの種類（上位5種）
    - filler_rate_per_min: 1分あたりのフィラー頻度
  - 教育的意味: フィラーや自己修復は「自分の発話を振り返るメタ認知の現れ」
    としてポジティブに評価する側面もある


================================================================================
■ ステップ4: LLM 評価（Anthropic Claude Sonnet 4）
================================================================================

書き起こしテキストを LLM に送信し、「ものさし」に基づく評価を取得します。

【システムプロンプトの動的構築】 build_system_prompt()
  学年段階に応じてプロンプトを動的に生成します。プロンプトには以下が含まれます:

  (A) 包括的なことばの発達ステージ（A〜F）の判定基準
      - 全学年段階で共通の基準
      - ステージ A（イマココ期）〜 F（評価・発展期）の6段階
      - 「聞く・話す」の観点から、思考・判断・表現を支える力を評価

  (B) 日本語の習得ステップ（1〜8）の判定基準
      - 学年段階ごとに異なる基準（STEP_RUBRIC から取得）
      - 日本語固有の知識・技能の習得状況を評価
      - 学年によって到達可能なステップ数が異なる
        （例: 小1〜小2 は最大ステップ5、小5〜中2 は最大ステップ8）

  (C) 追加のテキスト分析観点（4項目）
      - コミュニケーション方略（言い換え・迂回表現）
      - コードスイッチング（L1/L2 の混用パターン）
      - 相互行為能力（応答適切性・相槌）
      - 言語的創造性（オノマトペ・言葉遊び）

【API 呼び出し】 analyze_text_with_llm()
  - モデル: claude-sonnet-4-20250514
  - max_tokens: 2048
  - 入力: 書き起こしテキスト（ユーザーメッセージとして送信）
  - 出力: JSON 形式（正規表現でコードブロックを抽出 → パース）

【LLM の出力フィールド】
  {
    "stage":                     ステージ判定（A〜F）
    "stage_reasoning":           ステージ判定の根拠（2〜4文）
    "step":                      ステップ判定（1〜8）
    "step_reasoning":            ステップ判定の根拠（2〜4文）
    "communication_strategies":  コミュニケーション方略の分析
    "code_switching":            コードスイッチングの分析
    "interactive_competence":    相互行為能力の分析
    "linguistic_creativity":     言語的創造性の分析
    "strengths":                 強み・できていること
    "next_goals":                次の目標・改善点
    "support_suggestions":       指導・支援のアドバイス
  }

■ ステップ5: 結果表示（Streamlit UI）
================================================================================

解析結果を5つのセクションで表示します。

【セクション1: ステージ & ステップ判定】
  - 大きなバッジで「ステージ X」「ステップ Y」を表示
  - ステージインジケーター: A〜F のバーで現在位置をハイライト
  - ステップインジケーター: 1〜最大ステップのバーで現在位置をハイライト
  - 各判定の根拠テキストをフィードバックカードで表示

【セクション2: フィードバック（3カラム）】
  - 💪 強み・できていること（緑ボーダー）
  - 🎯 次の目標（オレンジボーダー）
  - 📖 支援のアドバイス（紫ボーダー）

【セクション3: 音声・認識モデルの分析（3カラム）】
  - 🎤 発音の自信度: 平均確信度 + 低確信度ワード一覧
  - 🔄 自己修復・フィラー: 回数・頻度・検出例
  - 😊 エンゲージメント: スコア + 声の傾向コメント

【セクション4: テキスト分析 LLM（2カラム）】
  - 💡 コミュニケーション方略
  - 🌐 コードスイッチング
  - 🤝 相互行為能力
  - ✨ 言語的創造性

【セクション5: 折りたたみセクション】
  - 🎙️ 音声分析の詳細データ:
    - レーダーチャート（流暢さ・ポーズ・発話速度・韻律の4軸）
    - 各種メトリクス（総時間・発話時間・発話比率・ポーズ回数 等）
  - 💬 文字起こしテキスト

■ レーダーチャートのスコア算出ロジック
================================================================================

音声分析の詳細データに表示されるレーダーチャートは、4つの軸で構成されます。
各軸のスコア（0〜100）は以下のロジックで算出されます。

  流暢さスコア = clip((phonation_time_ratio - 0.2) / 0.7 * 100, 0, 100)
    → 発話比率 0.2（20%）を基準に、0.9（90%）で満点

  ポーズスコア = clip(100 - length_penalty - freq_penalty, 0, 100)
    → length_penalty: 平均ポーズ長 0.3秒超過分のペナルティ（最大50点）
    → freq_penalty:   ポーズ頻度 10回/分超過分のペナルティ（最大50点）

  発話速度スコア = clip(chars_per_sec / 6.0 * 100, 0, 100)
    → 6文字/秒を満点基準（こどもの L2 学習者の適切範囲）

  韻律スコア = clip(pitch_cv / 0.30 * 100, 0, 100)
    → ピッチの変動係数（CV = std / mean）0.30 を満点基準
